# Мониторинг бэкапа конфигов и историй консолей (`config_console_backup_monitor`)

Расширение отслеживает результаты скрипта `scripts/copy_configs_ssh.sh`, который
запускается на **каждом** хосте и бэкап-сервере, собирает конфиги VM/LXC и истории
консолей и доставляет их **по SSH (rsync)** на приёмник (по умолчанию — бэкап-сервер,
как раньше; на бэкап-серверах приёмник переопределяется на NAS). Скрипт отправляет
**одно итоговое письмо на каждый прогон** на ящик `katok@202020.ru`; письмо фильтром
пересылается на почтовый сервер проекта (в Maildir), парсится
`ConfigConsoleBackupParserMixin` и сохраняется в таблицу `config_console_backups`
(одна запись на прогон).

## Что изменилось относительно старой схемы

Старая схема: хосты клали копии конфигов/историй на **смонтированную по сети шару**
бэкап-сервера, затем бэкап-сервер перекладывал их на NAS. Списки LXC-контейнеров
(`CONTAINER_IDS`) и пути их разделов прописывались в скрипте **вручную на каждом хосте**.

Новая схема (`copy_configs_ssh.sh`):

- **Доставка по SSH (rsync)** вместо сетевой шары. Хост/пользователь/путь приёмника и
  SSH-ключ настраиваются переменными (см. ниже).
- **Единый скрипт для всех машин** — байт-идентичен на хостах и бэкап-серверах.
  Различия задаются только в опциональном `/etc/copy_configs.conf`
  (см. `scripts/copy_configs.conf.example`).
- **Автоопределение гостей** через `pct list` (LXC) и `qm list` (VM) — ручной
  `CONTAINER_IDS` больше не нужен. На не-PVE машинах блок гостей пропускается.
- **Нетиповые subvol-пути** (`/rpool/subvol-141-disk-0`, `/rpool/subvol-142-disk-1`, …)
  больше не хардкодятся:
  - запущенные контейнеры читаются через `pct exec <id> -- cat …` (путь раздела не нужен);
  - остановленные — rootfs определяется из `pct config <id>` + `zfs get mountpoint`,
    с фолбэком на glob `/rpool/subvol-<id>-disk-*` (покрывает любой суффикс `-disk-N`).
- **Истории всех пользователей контейнера**: забираются `.bash_history` каждого
  пользователя (root, postgres, … — `/root` + `/home/*`), а также (опционально,
  `COLLECT_PSQL_HISTORY=1`) `.psql_history` (обычно у postgres).
- Конфиги хоста (`/etc/pve/lxc`, `/etc/pve/qemu-server`, `/opt`, `interfaces`, `fstab`,
  `main.cf`, `crontab`, host `.bash_history`) собираются как и раньше.

## Формат письма (его обязан соблюдать скрипт)

Тема (ASCII, не MIME-кодируется):

```
Config backup <host> <STATUS>
```

- `<host>` — короткое имя сервера (`sr-pve5`);
- `<STATUS>` — `OK` | `PARTIAL` | `ERROR`:
  - `ERROR` — не удалась доставка (rsync упал);
  - `PARTIAL` — доставка прошла, но часть элементов не собрана (`Ошибок > 0`);
  - `OK` — всё успешно.

Тело (UTF-8, по одному полю на строку, формат `Ключ: значение`, ключи в начале строки):

```
Хост: sr-pve5
Способ доставки: ssh-rsync
Приёмник: root@sr-bup:/bup/backup/sr-pve5
Начало: 2026-05-30 03:00:01
Завершено: 2026-05-30 03:02:17
VM конфигов: 4
LXC конфигов: 9
Контейнеров с историей: 9
Файлов истории: 17
Ошибок: 1
Проблемные элементы: lxc-164/postgres
```

- `Способ доставки:` / `Приёмник:` — произвольный текст (показывается как есть);
- `Начало:` / `Завершено:` — произвольный текст;
- `VM конфигов:` / `LXC конфигов:` / `Контейнеров с историей:` / `Файлов истории:` /
  `Ошибок:` — целые числа;
- `Проблемные элементы:` — список через запятую; строку можно опустить, если ошибок нет.

Статус берётся из темы. Проблема в интерфейсах подсвечивается, если
`STATUS = ERROR` или `Ошибок > 0`.

## Установка и настройка скрипта

1. Разложить `scripts/copy_configs_ssh.sh` в `/opt/copy_configs_ssh.sh` на всех хостах
   и бэкап-серверах (скрипт одинаковый везде).
2. Настроить **беспарольный SSH** до приёмника: приватный ключ (`SSH_KEY`) на отправителе,
   публичный — в `authorized_keys` приёмника. Реальные ключи в репозиторий не кладём.
3. При необходимости создать `/etc/copy_configs.conf` (по образцу
   `scripts/copy_configs.conf.example`). На **бэкап-серверах** в нём переопределяют
   приёмник на NAS:
   ```bash
   RECEIVER_HOST="nas-01"
   RECEIVER_BASE="/volume1/backup/configs"
   ```
4. Добавить в cron (пример — ежедневно в 03:00):
   ```
   0 3 * * * /opt/copy_configs_ssh.sh >/dev/null 2>&1
   ```

Переменные по умолчанию (переопределяются в `/etc/copy_configs.conf`):

| Переменная | Назначение | Дефолт |
|---|---|---|
| `RECEIVER_HOST` | хост-приёмник SSH | `sr-bup` |
| `RECEIVER_USER` | пользователь на приёмнике | `root` |
| `RECEIVER_BASE` | база на приёмнике (`$RECEIVER_BASE/<host>`) | `/bup/backup` |
| `SSH_KEY` | приватный ключ для rsync | `/root/.ssh/id_ed25519_backup` |
| `COLLECT_PSQL_HISTORY` | забирать `.psql_history` (1/0) | `1` |
| `MAIL_TO` | ящик отчётов | `katok@202020.ru` |
| `DRY_RUN` | не слать rsync/письмо, печать плана (1/0) | `0` |

Проверка без побочных эффектов: `DRY_RUN=1 DEBUG_LOG=/tmp/cc.log /opt/copy_configs_ssh.sh`.
