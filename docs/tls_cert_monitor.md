# Расширение `tls_cert_monitor` — мониторинг TLS-сертификатов

Расширение проверяет срок действия TLS-сертификатов доменов, позволяет
**перевыпускать их по кнопке** через `certbot` (по SSH на хосте с nginx) и
**загружать платный сертификат `202020.ru`** через Telegram-бот.

> Объём текущей итерации: бэкенд + Telegram. Отображение/настройка в Matrix-боте
> и Android-приложении — следующий этап (точки интеграции описаны ниже).

## Что делает

- **Проверка срока.** Для каждого включённого домена читает живой сертификат
  эндпоинта через `openssl s_client -connect <domain>:<port> -servername <domain>`.
  Это устойчиво к истёкшим/недоверенным сертификатам.
- **Алертинг.** Плановая проверка встроена в основной цикл мониторинга
  (`core/monitor_parts/lifecycle.py`). При приближении истечения (порог в днях,
  настраивается per-домен) уходит алерт; при возврате в норму — уведомление.
- **Ручной перевыпуск.** Кнопка в Telegram выполняет на удалённом хосте по SSH:

  ```sh
  certbot certonly --nginx --cert-name <domain> -d <domain> --force-renewal && service nginx restart
  ```

  SSH-хост и шаблоны команд (`certbot`, перезапуск nginx) настраиваются в боте.
- **Платный сертификат `202020.ru`.** Сертификат выдаётся провайдером и ставится
  вручную. Через бота можно загрузить файлы cert и key (как документы): они
  валидируются (`openssl`), проверяется соответствие пары и сохраняются в
  `data/certificates/202020.ru.{crt,key}`. Применение в nginx остаётся ручным.

## Домены по умолчанию

| Домен              | Порт  | Управление |
|--------------------|-------|------------|
| `api.202020.ru`    | 8443  | certbot    |
| `rtc.202020.ru`    | 443   | certbot    |
| `chat.202020.ru`   | 443   | certbot    |
| `share.202020.ru`  | 443   | certbot    |
| `matrix.202020.ru` | 443   | certbot    |
| `202020.ru`        | —     | платный (ручная загрузка) |

Список доменов редактируется в боте (добавить/удалить/вкл-выкл, порт, порог
алерта) или сбрасывается к значениям по умолчанию.

## Telegram

Главное меню → «🔐 TLS-сертификаты»:

- **🔄 Проверить сейчас** — опрос всех доменов и платного сертификата.
- **♻️ Перевыпуск (certbot)** — список доменов, перевыпуск по кнопке с
  подтверждением.
- **📜 Платный 202020.ru** — статус, загрузка cert/key документами, проверка пары.
- **⚙️ Настройки** — SSH-хост, команда `certbot`, команда перезапуска nginx,
  порог алерта по умолчанию, управление списком доменов.

## Настройки (хранятся в `settings.db` через `config_manager`)

- `TLS_CERT_DOMAINS` — словарь доменов `{domain: {enabled, port, alert_days}}`.
- `TLS_CERT_SETTINGS` — `{ssh_host, certbot_cmd, nginx_reload_cmd, alert_days_default, reissue_timeout}`.

`certbot_cmd` обязан содержать плейсхолдер `{domain}`.

## Требования

- На сервере мониторинга доступен бинарник `openssl` (проверка сроков и
  валидация загруженных файлов).
- SSH-доступ к хосту с `certbot`/`nginx` настроен через `SSH_KEY_PATH` /
  `SSH_USERNAME` (как у расширения свободного места ZFS).

## Точки интеграции (для будущих Matrix/Android)

- Ядро логики: `extensions/tls_cert_monitor.py` (чистые функции — переиспользуемы
  любым клиентом).
- Telegram: `bot/handlers/tls_cert_handlers.py`, диспетчер в
  `bot/handlers/callbacks.py`, кнопка в `bot/menu/builder.py` и
  `bot/menu/handlers.py`, ввод текста в
  `bot/handlers/settings_handlers/settings_value.py`, приём документа в
  `bot/handlers/__init__.py`.
- Matrix: добавить `ExtensionMenuItem` и запись в `_EXTENSION_SETTINGS`
  (`lib/matrix_commands.py`), хендлер вызывает `collect_certificates()` /
  `build_status_lines()`.
- Android: REST-эндпоинты `/v1/certificates/*` в
  `extensions/web_interface/__init__.py` поверх тех же функций ядра.
