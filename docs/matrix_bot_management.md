# Управление ботом в Matrix

## Что уже работает

В проекте уже реализована отправка уведомлений в Matrix через переменные окружения:

- `MATRIX_HOMESERVER`
- `MATRIX_ACCESS_TOKEN`
- `MATRIX_ROOM_ID`

Если любой из этих параметров пустой, канал Matrix считается отключённым.

## Быстрый запуск (минимум)

1. Создай отдельного Matrix-пользователя для бота (пример: `@monitoring-bot:matrix.202020.ru`).
2. Добавь его в нужную комнату уведомлений.
3. Получи `access token` этого пользователя.
4. Укажи env-переменные:

```bash
export MATRIX_HOMESERVER="https://matrix.202020.ru"
export MATRIX_ACCESS_TOKEN="<token_бота>"
export MATRIX_ROOM_ID='!roomid:matrix.202020.ru'
```

5. Перезапусти сервис мониторинга.

## Автоматическая настройка через скрипт

Для первичной инициализации можно использовать встроенный скрипт:

```bash
python scripts/setup_matrix_bot.py \
  --homeserver https://matrix.202020.ru \
  --username monitoring-bot \
  --password '<новый_пароль_бота>' \
  --room-id '!roomid:matrix.202020.ru'
```

Скрипт логинится, получает рабочий `MATRIX_ACCESS_TOKEN` и помогает проверить, что бот может писать в комнату.

## Операционное управление

### Проверки после деплоя

- Отправь тестовое уведомление из мониторинга и проверь доставку в Matrix-комнату.
- Убедись, что сообщения приходят и в Telegram, и в Matrix (если включены оба канала).
- Проверь, что `MATRIX_ROOM_ID` указан в полном формате (`!id:server`).

### Ротация токена

1. Выполни перевыпуск токена для bot-user.
2. Обнови `MATRIX_ACCESS_TOKEN` в секретах/окружении.
3. Перезапусти сервис.
4. Сразу проверь доставку тестового события.

### Типовые проблемы

- `M_FORBIDDEN`/403: токен просрочен или пользователь не имеет доступа к комнате.
- Сообщения не уходят: один из `MATRIX_*` параметров пустой.
- Бот «молчит» в комнате: bot-user не приглашён в room или исключён из неё.

## Команды и кнопки в чате (реализовано)

Входящие команды работают через `lib/matrix_commands.py` (long-polling `/sync`,
ACL по user-id и room-id, аудит каждой команды). Стек — `matrix-nio`.

### Текстовые команды

Пиши их прямо в комнату (бот ловит и инлайн в упоминаниях):

| Команда | Действие |
| --- | --- |
| `!menu` / `!start` / `!help` | Меню с кнопками-реакциями |
| `!status` | Доступность всех серверов |
| `!resources` | Ресурсы всех серверов (CPU/RAM/Disk) |
| `!report` | Утренний/сводный отчёт |
| `!servers` / `!list` | Список серверов под мониторингом |
| `!check <имя\|ip>` | Точечная проверка доступности сервера |
| `!res <имя\|ip>` | Ресурсы одного сервера (без аргумента — все) |
| `!pause` / `!resume` | Приостановить / возобновить мониторинг |
| `!silent` / `!loud` / `!auto` | Принудительно тихий / громкий / авто режим |
| `!mode` | Текущее состояние управления и расписание |
| `!about` | Версия и сведения о боте |
| `!settings get <KEY>` | Значение настройки |
| `!diag` / `!ping` | Диагностика command-bot |

### Кнопки (реакции)

На сообщение `!menu` бот навешивает emoji-реакции. Нажатие реакции в
**Element**, **Element X** или **web-клиенте** Matrix запускает
соответствующую команду (с теми же ACL и аудитом, что и текст):

`📡` статус · `📊` ресурсы · `🌅` отчёт · `🖥️` серверы ·
`⏸️` пауза · `▶️` resume · `🔇` тихо · `🔊` громко · `🔄` авто · `ℹ️` о боте

В Matrix нет нативных inline-кнопок как в Telegram, поэтому используются
реакции — единственный кросс-клиентский интерактив, который одинаково
работает в Element / Element X / web. Если сборка `matrix-nio` старая и не
отдаёт `ReactionEvent`, бот переходит на fallback через `UnknownEvent`;
текстовые команды работают всегда.

## Минимальный регламент эксплуатации

- Хранить токен только в секретах (не в git).
- Использовать отдельного bot-user (не личный аккаунт администратора).
- Ограничивать комнаты/права только нужными каналами.
- После каждого изменения токена или комнаты делать проверку тестовым алертом.
