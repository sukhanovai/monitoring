# План паритета Android-меню с Telegram

Цель: довести Android-клиент до функционального паритета с ключевыми разделами Telegram-бота, не ломая текущий backend-контур и контракт BFF.

## 1) Область паритета

### P0 (обязательно до релиза мобильного MVP)
1. Доступность всех серверов.
2. Управление мониторингом: pause/resume/report/quiet/loud.
3. Настройки monitoring.
4. Настройки bot.
5. Временные настройки (quiet window + metrics time).
6. Auth-настройки.

### P1 (вторая волна)
1. Проверка доступности одного выбранного сервера.
2. Ресурсы сервера (по выбранному серверу).
3. Экраны расширений (только включённые фичи).
4. Просмотр бэкапов (Proxmox/DB/mail/остатки) через единый раздел.
5. ZFS-раздел (read-only).

## 2) Текущее состояние

- ✅ В Android уже есть экран доступности всех серверов.
- ✅ Реализованы control actions (включая quiet/loud).
- ✅ Реализованы формы settings: monitoring/bot/time/auth.
- ✅ Добавлена базовая валидация форм: минимум одно поле и числовая проверка для `ssh_port` и monitoring-полей.
- ⚠️ Нужна фактическая проверка на стенде BFF и polishing UX (ошибки/подсказки/блокировки кнопок в loading).

## 4) Риски и ограничения

1. Контракты BFF частично в draft — возможны несовпадения полей.
2. Android build в CI/локально зависит от доступности AGP-артефактов и корректного JDK.
3. Для части Telegram-функций нужна серверная агрегация, которой пока нет в мобильном API.

## 5) Критерии готовности

1. Все P0-сценарии доступны из Android UI.
2. Ошибки API читаемы и содержат `request_id` в тексте/логах.
3. Нет регрессии Telegram-бота (тот же backend-контур, те же настройки).
